# `wave_reports/` bundle guide

This directory holds checked-in waveform bundles for promoted MuSiP DV cases.
Canonical case bundles are organized as `wave_reports/<bucket>/<case_id>/`:

- `BASIC/<case_id>/`
- `EDGE/<case_id>/`
- `PROF/<case_id>/`
- `ERROR/<case_id>/`
- `CROSS/<case_id>/`

Most promoted case bundles keep one same-axis VCD that records:

- `tb_top.feb_if0..3`
- `tb_top.opq_if`
- `tb_top.dma_if`
- `tb_top.ctrl_if`
- `tb_top.clk`

That VCD is the correlation source of truth for ingress, merged OPQ egress, and DMA.
The checked-in `packet_analyzer/` artifact is a static HTML packet-analyzer app rebuilt from case-local ledger artifacts; it serves `index.html`, `app.js`, `styles.css`, `packet-manifest.js`, and lazy lane payload scripts without depending on any mutable `external/` generator snapshot.
The same directory keeps the shared-axis WaveDrom ledger artifacts generated by `tb_int/scripts/generate_opq_twoframe_report.py`, so the served analyzer can correlate packet rows with ingress, merged OPQ egress, and DMA wave panels from the same captured VCD.
The promoted plain-harness bundles (`X113`, plus the `plain/` half of `X125`) use the same analyzer flow with the `plain` signal-layout mode, which decodes raw top-level buses under `tb_swb_block_plain_replay` instead of SystemVerilog interfaces.
`X125` is the one promoted dual-harness exception: it keeps `plain/` and `twoenv/` subbundles plus a top-level `compare_summary.txt`.
Physical waveform evidence uses the promoted `N_SHD=128` frame cadence, so adjacent frame starts are spaced by `4096` cycles at the `250 MHz` harness clock unless a bundle is explicitly labeled as compressed visualization evidence.

## Serve

Serve the full report root with:

```bash
python3 -m http.server 8765 --bind 127.0.0.1 --directory tb_int/wave_reports
```

Serve any bundle's packet analyzer with:

```bash
python3 /home/yifeng/packages/mu3e_ip_dev/mu3e-ip-cores/tools/packet_transaction_traffic_analyzer/scripts/serve_packet_analyzer.py \
  --dir tb_int/wave_reports/BASIC/B047/packet_analyzer \
  --port 8765
```

## Navigation help

- Use the ingress panel to verify the frame-origin timestamp contract and the enabled-lane mask on the captured lanes.
- Use the merged OPQ egress panel to inspect the actual merged frame stream that corresponds to the same ingress window.
- Use the DMA panel to correlate packed 256-bit payload words back to the selected ingress/egress frames.
- Open the lazy-loaded ledgers below each panel when you need the exact cycle/time/data rows rather than the compressed waveform view.

## Correlation note

The served `packet_analyzer/` page is already the ingress/egress/DMA correlation view for the case-local VCD.
`bundle.json` also records the exact interface-role mapping:

- `tb_top.feb_if0..3` = ingress lanes
- `tb_top.opq_if` = merged OPQ egress
- `tb_top.dma_if` = DMA payload sink
- `tb_top.ctrl_if` = reset / DMA / mask control

The masked-lane BASIC bundles intentionally keep all four ingress interfaces in the VCD even when only one lane survives the musip-local `feb_enable_mask`, so the pre-mask source traffic and the post-merge OPQ/DMA result stay visible in one recording.
